## 用户需求

用户要求对公众号AI写作项目进行全面审查，然后启动服务并打开预览供用户查看。

## 审查范围

1. **代码质量审查**：检查TypeScript类型、React最佳实践、组件结构、状态管理
2. **架构一致性**：验证三服务架构（DailyHotApi、Server、Frontend）的协调性
3. **依赖关系**：检查package.json依赖版本、安全漏洞、重复依赖
4. **潜在问题**：内存泄漏、未处理错误、性能瓶颈、git状态中的删除文件影响
5. **启动流程**：确认所有服务能正常启动，端口配置正确

## 预览目标

成功启动所有服务，并打开http://localhost:5173供用户预览项目效果

## 技术方案

### 审查方法

1. **静态分析**：使用code-review-expert skill进行代码质量审查
2. **依赖检查**：分析package.json和package-lock.json的依赖关系
3. **架构验证**：检查服务间通信、API路由、代理配置
4. **运行时检查**：启动服务验证功能完整性

### 启动策略

1. **顺序启动**：DailyHotApi (6688) → Server (6356) → Frontend (5173)
2. **健康检查**：每个服务启动后验证健康状态
3. **错误处理**：服务启动失败时的回退方案

### 预览方案

使用gstack skill进行浏览器自动化，打开http://localhost:5173并验证页面加载

## 可用扩展

### Skill

- **code-review-expert**
- Purpose: 对项目代码进行全面审查，检测SOLID违规、安全风险，提供改进建议
- Expected outcome: 生成详细的代码审查报告，包括问题清单和改进建议

- **gstack**
- Purpose: 使用浏览器自动化工具打开预览页面，验证项目运行状态
- Expected outcome: 成功打开http://localhost:5173，截取页面截图供用户查看

- **systematic-debugging**
- Purpose: 系统化调试方法，用于排查服务启动和运行中的问题
- Expected outcome: 快速定位并解决项目中的技术问题
